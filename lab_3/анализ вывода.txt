migration
	В компах нет единаго модуля, который бы определял загруженной каждго ядра и перераспределял между ними работу. Это слишом большой обьем анализируемой информации, причем выполняемый постоянно - слишком затратно.
Поэтому существует **миграция**. Каждое ядро скорее всего имеет ограничение на очередь выполняемых процессов. 	Если очередь переполняется происходит миграция, т е процесс переходит на выполнение на другое ядро.
	Процесс миграции ядра распределяет рабочую нагрузку между ядрами процессора. На каждое ядро процессора должен приходиться один процесс миграции.

ksmd
Объединение одинаковых страниц ядра (KSM) — это функция дедупликации анонимной памяти для экономии памяти. 
Общий подход заключается в совместном использовании анонимных страниц разных процессов с одинаковым содержимым. Вместо нескольких страниц с одинаковым содержимым создаётся страница KSM, которая используется всеми процессами. Это достигается путём указания на эту страницу KSM в записях таблицы страниц всех страниц. Страница KSM помечается как «копирование при записи» (COW). Если один из процессов, использующих эту страницу, захочет её изменить, создаётся дубликат страницы, который процесс может изменить.
